     1 ОСНОВНІ РЕКОМЕНДАЦІЇ НАПИСАННЯ КОДУ НА МОВІ ПРОГРАМУВАННЯ SQL
     
     1.1 Мета роботи 
     
     Основна мета полягає у засвоєнні ключових принципів написання чистого, ефективного й зручного для підтримки коду в різних мовах програмування. Крім того, передбачається набуття навичок аналізу та рефакторингу програмного коду з метою підвищення його якості й зрозумілості.
     
     1.2 Хід роботи 
     
1. Для вивчення правил оформлення коду обрано мову SQL і проаналізовано низку джерел, що стосуються найкращих практик написання SQL-запитів.
2. Підготовано фрагменти SQL-коду для ілюстрації ключових принципів.
3. Створено презентацію (див. дод. А), у якій розглянуто такі основні аспекти:
○ важливість правил оформлення коду;
○ структура SQL-запитів;
○ форматування коду (застосування відступів, великі/малі літери для ключових слів);
○ іменування об’єктів (таблиць, стовпців, псевдонімів);
○ коментарі в SQL;
○ документування коду;
○ конвенції стилю для різних СУБД;
○ тестування SQL (включно з концепцією TDD там, де можливо);
○ інструменти та методи автоматизації для контролю якості SQL-коду.
4. Оформлено звіт до практичного заняття.

     1.3 Висновок
     
     Під час виконання роботи було детально вивчено основні принципи написання чистого, ефективного та легкого у підтримці коду мовою SQL. Особливу увагу приділено аналізу й рефакторингу наявних запитів, зокрема дотриманню стилю, оптимальному іменуванню таблиць і стовпців, правильному форматуванню, а також упровадженню методології тестування (зокрема TDD) та автоматизації процесів для підвищення якості коду. Ці підходи допомагають вчасно виявляти потенційні проблеми та забезпечувати високу підтримуваність рішень, що використовують бази даних. Створення презентації дозволило узагальнити напрацьовані знання й підкреслити важливість аспектів на кшталт документування коду та застосування інструментів перевірки стилю, без чого складно гарантувати якісні SQL-рішення.
